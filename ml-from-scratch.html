01 — Overview

Project Overview

Machine learning libraries abstract away the mathematics that make them work. This project removes those abstractions entirely. Every algorithm is derived from its mathematical definition and implemented using only NumPy — no autograd, no sklearn, no shortcuts.

The philosophy: if you can derive the gradient update rule by hand, you understand the algorithm. If you can only call .fit(), you're using a tool you don't fully understand. This project proves the former.

03 — Mathematical Background

The Mathematics

# Gradient Descent update rule
θ ← θ - α · ∇_θ L(θ)

# Logistic Regression — sigmoid + log-loss
σ(z) = 1 / (1 + e⁻ᶻ)
L = -[y log(ŷ) + (1-y) log(1-ŷ)]
∇_θ L = (1/m) Xᵀ(σ(Xθ) - y)

# Backprop — chain rule applied layer by layer
∂L/∂W¹ = ∂L/∂a² · ∂a²/∂z² · ∂z²/∂a¹ · ∂a¹/∂z¹ · ∂z¹/∂W¹
04 — Implementation

Implementation Approach

  1. Derive first, code second — Every gradient expression is written out analytically on paper before being implemented in NumPy.

  2. Gradient checking — Numerical gradient approximation (finite differences) used to verify analytical gradients are correct: |∇analytical - ∇numerical| < 1e-7.

  3. Loss curve logging — Every training run logs cost per iteration; convergence behavior inspected for learning rate sensitivity.

  4. sklearn benchmark — Final model weights and predictions compared to equivalent sklearn model on identical dataset.

import numpy as np class LogisticRegression: """Binary classifier via maximum likelihood estimation.""" def __init__(self, lr=0.01, n_iter=1000, lam=0.0): self.lr = lr; self.n_iter = n_iter; self.lam = lam def _sigmoid(self, z): return 1 / (1 + np.exp(-np.clip(z, -500, 500))) def fit(self, X, y): m, n = X.shape self.theta_ = np.zeros(n + 1) X_b = np.column_stack([np.ones(m), X]) self.losses_ = [] for _ in range(self.n_iter): y_hat = self._sigmoid(X_b @ self.theta_) # Binary cross-entropy loss + L2 regularization loss = -np.mean(y * np.log(y_hat + 1e-15) + (1 - y) * np.log(1 - y_hat + 1e-15)) loss += (self.lam / (2*m)) * (self.theta_[1:] @ self.theta_[1:]) self.losses_.append(loss) # Gradient: ∂L/∂θ = (1/m) Xᵀ(σ(Xθ) - y) + λθ/m grad = (X_b.T @ (y_hat - y)) / m grad[1:] += (self.lam / m) * self.theta_[1:] self.theta_ -= self.lr * grad return self def predict_proba(self, X): X_b = np.column_stack([np.ones(len(X)), X]) return self._sigmoid(X_b @ self.theta_) def predict(self, X, threshold=0.5): return (self.predict_proba(X) >= threshold).astype(int)

06 — Insights

Key Insights

  • Gradient descent is sensitive to learning rate by orders of magnitude. Too large → divergence. Too small → impractically slow convergence. Learning rate schedules matter.
  • Backpropagation is just the chain rule applied repeatedly. Once that clicks, neural networks stop feeling like magic.
  • L2 regularization has a clean geometric interpretation: it penalizes the ℓ² norm of weights, shrinking all coefficients proportionally. L1 produces sparsity by shrinking small weights to exactly zero.
  • Mini-batch gradient descent is a principled compromise: it reduces gradient variance without requiring the full dataset, making it both fast and stable.
